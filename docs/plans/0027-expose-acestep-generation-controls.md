# Expose ACE-Step Generation Controls Plan

## Problem

ACE-Step repaint quality depends on parameters that are currently fixed in code. That makes the UI easier at first, but it blocks users from tuning the controls that matter when a repaint comes out garbled, too conservative, too loose, too slow, or poorly stitched.

## Approach

Add a compact advanced settings area to the Transition panel. Keep the normal workflow simple by pre-filling model-aware defaults, while allowing direct user overrides for ACE-Step repaint and inference parameters.

Also make generated outputs usable inside the app: results should appear as a history list with playable audio, not just a path readout.

ACE-Step repaint supports selective editing/regeneration of an existing source region, not only extending into empty appended space. Add a generation-region choice so users can either extend into blank space after the selected marker or repaint over the existing "new section" region after the marker when enough source audio exists.

## Controls To Add

- Inference steps: numeric input. Defaults to the selected model profile's default steps.
- Guidance scale: numeric input. Relevant mainly for base/non-turbo models.
- Shift: numeric input. Relevant mainly for base/non-turbo models.
- Repaint mask mode: select with `explicit` as default and `auto` available for experimentation.
- Repaint mode: select with `balanced`, `conservative`, and `aggressive`.
- Repaint strength: numeric input from `0` to `1`.
- Latent crossfade frames: numeric input.
- Wave crossfade seconds: numeric input, even though current ACE-Step runtime behavior may partially derive this from mode/strength internally.
- Generation region: segmented control or select with:
  - `Extend after marker`: current behavior, scaffold is source tail plus blank continuation.
  - `Repaint existing audio`: use the source audio after the marker as the repaint target instead of appending blank space. Only enable when the selected source has enough audio after the marker for the configured new-section length.

## API/Data Changes

- Add optional fields to `GenerateSelectionRequest`.
- Add optional repaint settings to `SourceSelectionPlan` metadata so each output records the exact tuning used.
- Add a generation-region field to the request/plan metadata so outputs record whether they were generated by blank extension or repainting existing audio.
- Pass only provided user overrides through to `AceStepApiClient.repaint`; otherwise use model-aware defaults.
- Return the effective ACE-Step payload fields in generated metadata where practical, without leaking large or irrelevant runtime data.
- Add a result model/API shape that can return multiple recent generated candidates with audio URLs/paths, status, model, prompt, timing, and metadata path.

## UI Changes

- Add an "ACE-Step Advanced" block under the current timing/BPM/seed controls.
- Add a generation-region control near the timing settings, because it changes what the selected marker means.
- Use clear labels with units where relevant.
- Keep the controls visually compact so the main source-selection workflow stays prominent.
- Add a Reset button that reapplies the selected model defaults.
- When the selected model changes, update defaults unless the user has manually edited advanced values.
- Replace the single generated path display with a results list. Each generated item should include a playable audio control, status, model name, prompt summary, timing summary, output path, metadata path, and a clear current/latest marker.

## Tests

- API tests for default payload behavior.
- API tests for user override payload behavior.
- UI/API tests that generation accepts advanced settings and records them in plan metadata.
- Tests for generation-region behavior:
  - blank extension keeps appending silence after the selected tail.
  - existing-audio repaint uses source audio after the marker and rejects configurations that exceed the available source duration.
- UI/API tests that generated outputs appear in the returned results list with playable audio paths.

## Risks

- Too many knobs can make the UI feel intimidating, so the section should be collapsed or visually secondary.
- ACE-Step may ignore or reinterpret some parameters depending on model/runtime version; labels should avoid promising exact behavior.
- Model defaults need to remain the fallback so casual users can still generate without knowing these controls.
- Existing-audio repaint changes source-scaffold construction and needs careful validation so users do not accidentally repaint beyond the available loaded song.
